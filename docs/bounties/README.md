# ResearchFlow Bug Bounty Program

## Overview

We value the security research community and welcome responsible disclosure of security vulnerabilities in ResearchFlow.

## Scope

### In Scope

- ResearchFlow web application
- API endpoints
- Authentication/authorization systems
- PHI handling and governance
- Data storage and encryption

### Out of Scope

- Third-party dependencies (report to upstream)
- Social engineering attacks
- Physical security
- Denial of service attacks

## Severity Levels

| Level | Description | Reward Range |
|-------|-------------|--------------|
| Critical | PHI exposure, auth bypass, RCE | $1,000 - $5,000 |
| High | Privilege escalation, data leakage | $500 - $1,000 |
| Medium | XSS, CSRF, information disclosure | $100 - $500 |
| Low | Minor issues, best practice violations | Recognition |

## Submission Process

1. **Discovery**: Document the vulnerability with reproduction steps
2. **Report**: Email security@researchflow.example.com with:
   - Detailed description
   - Steps to reproduce
   - Potential impact assessment
   - Suggested remediation (optional)
3. **Response**: We will acknowledge within 48 hours
4. **Resolution**: We aim to fix critical issues within 7 days
5. **Disclosure**: Coordinated disclosure after patch is released

## Rules of Engagement

- Do not access or modify data belonging to other users
- Do not perform testing that degrades service availability
- Do not publicly disclose before coordinated timeline
- Provide sufficient detail for reproduction
- Act in good faith

## Recognition

Security researchers who report valid vulnerabilities will be:

- Listed in our Security Hall of Fame (with permission)
- Eligible for monetary rewards based on severity
- Given credit in security advisories

## Safe Harbor

We will not pursue legal action against researchers who:

- Follow responsible disclosure guidelines
- Act in good faith
- Do not cause harm to users or systems
- Report findings promptly
